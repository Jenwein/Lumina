# Lumina 项目详细需求文档 (PRD)

## 1. 项目概述
`Lumina` 旨在打破 AI 助手与桌面环境之间的隔阂。通过将大语言模型 (LLM) 的推理能力与游戏引擎渲染的桌面宠物相结合，创建一个既具备强大办公/操作能力，又具有情感价值和视觉反馈的交互式 Agent。

## 2. 核心模块与功能需求

### 2.1 Agent 决策引擎 (Lumina Core - Python)
*   **多模型集成：**
    *   支持所有兼容 OpenAI 格式的 API（GPT-4, Claude, DeepSeek, Ollama 等）。
    *   用户可自由配置多个 API 节点并随时切换。
*   **ReAct 自主循环：**
    *   **思考 (Thought)：** 分析用户需求，拆解步骤。
    *   **行动 (Action)：** 选择并调用相应的工具。
    *   **观察 (Observation)：** 接收工具返回的结果（如 OCR 坐标、网页内容）并评估是否达成目标。
*   **角色与记忆：**
    *   支持系统提示词自定义，赋予桌宠独特的性格。
    *   维护对话历史，确保任务执行的连续性。

### 2.2 工具系统 (The Hands)
*   **系统控制工具：** 文件管理（增删改查）、应用启动/关闭、系统状态监测。
*   **Web 自动化工具：** 基于 Playwright 实现静默或可见的浏览器操作。
*   **全场景视觉模拟工具：**
    *   **屏幕感知：** 截图并执行 OCR (文字识别) 和图标检测。
    *   **通用交互：** 模拟点击、拖拽、组合键输入，实现对无 API 应用的控制。

### 2.3 桌宠表现层 (Lumina UI - Godot)
*   **窗口特性：** 背景透明、全屏置顶、支持鼠标穿透（动态切换）。
*   **动画系统：**
    *   **状态机：** 待机、移动、思考中、打字中、观察中（视觉分析时）、点击动作、庆祝/失败。
    *   **跟随机制：** 当 Agent 准备执行点击时，桌宠需平滑移动至目标坐标，播放点击动画后再触发实际的物理点击。
*   **UI 交互：**
    *   **对话气泡：** 替代传统的对话框。
    *   **右键菜单：** 提供设置、模型切换、日志查看及紧急停止按钮。

## 3. 技术路线图与架构

*   **架构模式：** 客户端-服务器架构。Python 后端作为服务端（Logic Server），Godot 前端作为客户端（Display Client）。
*   **通信协议：** 本地 WebSocket (端口可配置)，传输 JSON 序列化指令。
*   **跨平台方案：**
    *   **Windows:** 原生支持。
    *   **macOS:** 针对沙盒权限和录屏授权进行引导提示。
    *   **Linux:** 同时支持 X11 和 Wayland（Wayland 下截图将适配管道流或 XDG Desktop Portal）。

## 4. 特殊场景与安全需求

### 4.1 多显示器支持
*   **需求：** Agent 必须能够识别并操作所有连接的显示器。
*   **实现：** Python 后端需获取所有屏幕的统一坐标系映射，并将坐标同步给 Godot，确保桌宠能跨屏“跑动”。

### 4.2 安全与确认机制
*   **需求：** 保护用户系统免受意外破坏。
*   **实现：** 
    *   引入“高危操作拦截”：涉及删除文件、格式化、执行敏感终端命令时，桌宠必须弹出一个醒目的确认气泡，待用户物理点击桌宠确认后方可执行。
    *   提供“单步执行”模式供用户调试。

### 4.3 资源优化
*   **需求：** 减少对日常工作的干扰。
*   **实现：** 
    *   **按需 OCR：** 仅在 Agent 无法通过常规 API 获取信息时才触发截图和视觉分析。
    *   **低功耗模式：** 桌宠在闲置时降低帧率。

### 4.4 隐私保护
*   **需求：** 防止敏感数据泄露给 LLM。
*   **实现：**
    *   **黑名单应用：** 用户可配置特定程序（如银行、密码管理器），当这些程序处于前台时，Agent 自动禁用截图功能。
    *   **隐私遮罩：** 允许用户自定义屏幕上的“不准看”区域。

### 4.5 模型资产计划
*   **阶段一：** 采用 2D 序列帧或 SpriteSheet（快速原型验证）。
*   **阶段二：** 引入 Spine 或 Live2D SDK，实现更生动的二次元角色。
*   **阶段三：** 支持 VRM 等 3D 模型格式，利用 Godot 的 3D 渲染能力。

## 5. 开发里程碑

1.  **里程碑 1 (V0.1):** 完成 Python 与 Godot 的 WebSocket 连通，实现桌宠在屏幕内根据指令跑动。
2.  **里程碑 2 (V0.2):** 实现 Agent 基础对话及系统文件操作工具。
3.  **里程碑 3 (V0.3):** 集成屏幕截图与 OCR，实现“桌宠跑过去点击按钮”的完整链路。
4.  **里程碑 4 (V0.4):** 跨平台适配与多模型配置界面开发。
5.  **里程碑 5 (V1.0):** 正式发布，包含完善的交互动画、安全机制和用户文档。
